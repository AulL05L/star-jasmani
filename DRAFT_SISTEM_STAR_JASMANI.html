1. DESKRIPSI UMUM SISTEM

Star Jasmani Digital Assessment adalah aplikasi web berbasis Laravel yang dirancang untuk memdigitalisasi proses penilaian dan pemantauan kebugaran jasmani (Samapta) bagi atlet/peserta yang mempersiapkan diri mengikuti seleksi instansi kedinasan (POLRI, TNI-AD/AL/AU).

Sistem ini menggantikan metode pencatatan manual berbasis spreadsheet Excel yang selama ini digunakan oleh pelatih (Coach). Dengan sistem ini, Coach dapat menginput nilai secara real-time di lapangan menggunakan perangkat apapun (smartphone/tablet), sementara setiap atlet dapat memantau tren perkembangan fisiknya sendiri secara mandiri melalui portal pribadi.

1.1 Tujuan Utama Sistem

  • Digitalisasi rekam data penilaian Samapta seluruh atlet secara terstruktur per-angkatan (Batch) dan per-parameter sesi
  • Kalkulasi otomatis nilai akhir kelulusan berdasarkan standar formula POLRI
  • Visualisasi tren performa individu dan kelompok melalui dashboard grafik interaktif
  • Generasi laporan resmi (PDF berlogo Star Jasmani) yang dapat dicetak atau diunduh
  • Arsitektur database extensible untuk mendukung multi-instansi (TNI-AD/AL/AU) di masa depan tanpa perubahan kode

1.2 Target Pengguna

RoleSebutanHak Akses

4. LOGIKA KALKULASI SKOR (STANDAR POLRI)

4.1 Formula Nilai Akhir Kelulusan

Jasmani A = Nilai Lari 12 Menit

Jasmani B = (Nilai Push-Up + Nilai Sit-Up + Nilai Pull-Up/Chin-Up + Nilai Shuttle Run) ÷ 4

Nilai Jasmani = (Jasmani A + Jasmani B) ÷ 2

Nilai Akhir = (Nilai Jasmani × 80%) + (Nilai Renang × 20%)

Contoh perhitungan: Lari 3.000m (Skor 80) | Push-Up 30 (80) | Sit-Up 35 (85) | Pull-Up 10 (80) | Shuttle 12.0s (80) | Renang skor 80

Jasmani B = (80 + 85 + 80 + 80) ÷ 4 = 81,25
Nilai Jasmani = (80 + 81,25) ÷ 2 = 80,63
Nilai Akhir = (80,63 × 80%) + (80 × 20%) = 64,50 + 16,00 = 80,50 → Grade A

5. STRUKTUR PERIODE & MANAJEMEN ANGKATAN (BATCHING)

5.1 Konsep Periodisasi Tes

Dalam satu tahun berjalan, sistem menampung 2 hingga 4 Parameter sesi tes yang berjalan secara berkala. Setiap parameter mewakili satu siklus penilaian lengkap (semua item fisik diukur ulang).

TahunParameterKeterangan
2025Parameter 1Sesi pertama awal tahun
2025Parameter 2Sesi kedua pertengahan tahun
2025Parameter 3Sesi ketiga akhir tahun
2025Parameter 4 (opsional)Sesi keempat jika diperlukan

5.2 Manajemen Kelompok (Batch)

Setiap atlet tergabung dalam satu angkatan (Batch). Data penilaian dan laporan rekapitulasi dikelompokkan per-angkatan agar rekam data tidak bercampur antar generasi peserta.

6. MODUL FITUR SISTEM

6.1 Modul Penilaian Samapta Kedinasan

  • Input nilai 6 item tes: Lari, Push-Up, Sit-Up, Pull-Up/Chin-Up, Shuttle Run, Renang
  • Kalkulasi otomatis sesuai formula POLRI (Jasmani A, B, Total, Final)
  • Filter rekap penilaian berdasarkan Batch dan Grade
  • Riwayat lengkap semua sesi tes per atlet
  • Form validation ketat (tipe data, batas nilai, otorisasi Coach)

6.2 Modul Kebugaran Umum (General Fitness)

  • Input Berat Badan (kg) dan Tinggi Badan (cm)
  • Kalkulasi otomatis BMI: BMI = BB ÷ TB² (meter)
  • Status otomatis: Kurang (BMI <17), Normal (17–24,9), Gemuk (25–29,9), Obesitas (≥30)
  • Riwayat rekam BMI tersimpan per tanggal pengukuran

6.3 Modul Riwayat Cedera (Injury Track)

  • Coach menginput catatan cedera secara manual (teks bebas)
  • Data yang dicatat: jenis cedera, tanggal, catatan pemulihan, status pulih/aktif
  • Ditampilkan sebagai informasi medis di dashboard atlet
  • Penting: Fitur ini 100% tidak mempengaruhi kalkulasi nilai akhir Samapta

6.4 Modul Laporan PDF

  • Generate laporan resmi PDF berlogo dan identitas "Star Jasmani"
  • Format landscape A4, font DejaVu Sans
  • Tersedia untuk: Admin (generate semua atlet), Member (download laporan pribadi)
  • Nama file otomatis: Laporan-Samapta-[Nama Atlet]-[Tanggal].pdf

8. KEAMANAN SISTEM

AspekImplementasi
SQL Injection100% menggunakan Eloquent ORM / Query Builder Laravel — tidak ada raw SQL langsung dari input pengguna
AutentikasiLaravel session-based auth dengan bcrypt hashing (12 rounds) untuk password
Rate LimitingMaksimal 5 percobaan login per menit per IP + email. Lockout otomatis dengan countdown.
Akses KontrolRoleMiddleware memvalidasi role setiap request. Member yang mencoba akses rute Admin mendapat HTTP 403.
Data PrivasiMember hanya bisa query data dengan where('user_id', Auth::id()) — tidak bisa akses data atlet lain
Session SecuritySession regenerate setelah login sukses untuk mencegah session fixation
Soft DeleteData users/athletes/scores tidak benar-benar terhapus — tercatat di kolom deleted_at untuk audit trail
Form ValidationSemua input divalidasi via FormRequest (StoreSamaptaRequest) sebelum masuk ke database
